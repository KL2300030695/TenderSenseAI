# **App Name**: Tender Sense AI

## Core Features:

- Secure Document Upload: Allow users to securely upload tender and bidder documents (PDFs, images) for processing. Includes document organization and version control.
- AI-Powered Criteria Extraction: Utilize a large language model as a tool to extract and categorize all eligibility criteria (technical, financial, compliance, mandatory/optional) from uploaded tender documents.
- AI-Powered Bidder Data Extraction: Employ OCR and NLP to automatically extract structured data (e.g., turnover, certifications, project experience) from bidder submission documents, along with original text snippets as evidence.
- Automated Evaluation Engine: Automatically compare extracted bidder data against tender criteria, classifying each as 'Eligible', 'Not Eligible', or 'Needs Manual Review' based on strict matching rules.
- Explainable Decision Reporting: Generate detailed, human-readable explanations as a tool for each evaluation decision, providing the criterion, required value, bidder value, status, confidence score, and clear source document references.
- Human-in-the-Loop Review Interface: Provide a user interface for human reviewers to address 'Needs Manual Review' cases, approve, edit, or reject AI-generated evaluations, and mark potential risks or inconsistencies.
- Consolidated Evaluation Dashboard: Display a dashboard with consolidated tender evaluation results, including eligible, rejected, and review-pending bidders, and generate audit-ready, exportable reports.

## Style Guidelines:

- Primary color: A deep, professional blue (#28518A), evoking trust and government-grade reliability.
- Background color: A very light, desaturated blue-grey (#F0F2F4), providing a clean and serene backdrop for detailed information.
- Accent color: A soft, muted cyan (#9CC6C9), offering a complementary visual cue for highlights and interactive elements while maintaining a professional aesthetic.
- Headline and body font: 'Inter', a modern and highly legible sans-serif, ensuring clarity and consistency across detailed reports and interface elements for an audit-ready platform.
- Use a set of clear, concise, and professional vector icons to represent document types, eligibility statuses (e.g., checkmarks, crosses), review actions, and data visualizations, ensuring easy comprehension at a glance.
- Implement a structured, data-centric layout with clean lines, ample whitespace, and clear hierarchy. Information should be presented in tables, cards, and distinct sections, optimizing for readability of complex data and reports.
- Incorporate subtle and functional animations for feedback during document uploads, processing statuses, loading states, and state changes (e.g., eligibility toggles) to enhance user experience without distraction.
